# IP Cleanup — Флаги для ручных действий

Файлы и ресурсы, которые нельзя вычистить автоматически. Требуется ручное вмешательство.

---

## 1. `assets/images/screen-gameplay.png` — визуальный контент

**Проблема:** нельзя автоматически проверить, содержит ли скриншот элементы оригинальной игры Namco
(стартовый экран с надписью "BATTLE CITY", меню "1 PLAYER / 2 PLAYERS / CONSTRUCTION", шрифт Namco).

**Что нужно сделать вручную:**
- Открыть файл и визуально убедиться, что на скриншоте виден только геймплей (танки, стены, базы),
  а не стартовый экран или UI, скопированный с оригинала Namco.
- Если на скриншоте присутствует UI оригинала — заменить на нейтральный скриншот геймплея или
  временную заглушку (`assets/images/screen-gameplay-placeholder.png`).

**Текущий статус:** скриншот используется в секции "Our Games" на главной странице.

---

## 2. `docs/TZ_Site_IP_Cleanup.md` — планировочный документ

**Проблема:** файл содержит ~40 упоминаний "Battle City", "Namco", "Bandai Namco", "NES" — но все они
являются частью анализа рисков и описания того, что нужно вычистить. Это внутренний рабочий документ,
не публикуемый на сайте напрямую.

**Решение:** файл оставлен без изменений. Он не отображается в навигации сайта и не индексируется
как пользовательская страница. Если GitHub Pages доступен публично — рассмотреть перенос папки `docs/`
за пределы публичного корня или добавить в `robots.txt` директиву `Disallow: /docs/`.

**Что нужно сделать вручную (опционально):**
- Добавить `robots.txt` с `Disallow: /docs/` чтобы поисковики не индексировали внутренние ТЗ.

---

## 3. Репозиторий на GitHub — название и описание

**Проблема:** если репозиторий на GitHub называется `battle-city-website` или содержит "Battle City"
в описании — это видно публично.

**Что нужно сделать вручную:**
- Зайти в Settings репозитория на GitHub.
- Переименовать репозиторий в `petrovbit.dev` или `petrovbit-website`.
- Обновить описание: "PetrovBit indie game studio website".

---

## 4. Ссылки на магазины — package ID

`io.battle_city.game` присутствует в URL ссылок на Google Play и RuStore. Это технический
идентификатор приложения — его нельзя изменить без перепубликации приложения под новым package ID.

**Статус:** оставлен как есть в href-атрибутах кнопок. Это допустимо — URL не отображается пользователю
как текст.
